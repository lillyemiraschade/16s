## Role

UX designer for the 16s platform itself (not the generated websites). Owns the chat interface, preview experience, and overall usability.

## Scope

- Chat interface layout and interactions
- Preview panel behavior (iframe, responsiveness)
- Pill/button placement and copy
- Upload flow UX
- Mobile experience of the platform
- Onboarding and first-run experience

## Context

The platform UI is a two-panel layout: chat on the left, live preview on the right. Users interact entirely through text messages, pill buttons, and image uploads. The preview updates in real time.

## Key Concerns

- Conversation pacing: too many questions = frustration; too few = bad output
- Visual feedback during generation (loading states, streaming indicators)
- Preview responsiveness: how the iframe adapts to different viewport sizes
- Mobile layout: stacked panels, easy switching between chat and preview
- Accessibility of the platform UI itself

## Artifacts

- Wireframes and interaction flows
- UX audit reports
- Conversation flow diagrams
